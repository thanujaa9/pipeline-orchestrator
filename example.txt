## Case 1 — TTL Expiry Demo

```bash
# Step 1 — Update src/config/queues.ts — add TTL back
# Change urgent options to:
'x-message-ttl': 10000,  # 10 seconds only

# Step 2 — Delete and recreate queues
docker exec -it rabbitmq rabbitmqadmin delete queue name=urgent.queue --username=admin --password=password
docker exec -it rabbitmq rabbitmqadmin delete queue name=batch.queue --username=admin --password=password
docker exec -it rabbitmq rabbitmqadmin delete queue name=dead.letter.queue --username=admin --password=password

# Step 3 — Start orchestrator only (NO consumers)
npx ts-node src/orchestrator/orchestrator.ts

# Step 4 — Start API
npx ts-node src/api/server.ts

# Step 5 — Send messages (no consumers running = messages will expire)
npx ts-node src/benchmark/bulkProducer.ts

# Step 6 — Wait 10 seconds then check
docker exec -it rabbitmq rabbitmqadmin list queues --username=admin --password=password
# dead.letter.queue → filling up ✅

# Explain to team:
# "Messages expired after 10s because no consumer was fast enough
#  RabbitMQ automatically moved them to DLQ"
```

## Case 2 — Broken Message (Consumer Rejection) Demo

```bash
# Step 1 — Remove TTL from src/config/queues.ts (back to no TTL)
# Delete queues and restart orchestrator

# Step 2 — Start everything
npx ts-node src/orchestrator/orchestrator.ts  # T1
npx ts-node src/api/server.ts                 # T2
npx ts-node src/consumers/consumerManager.ts  # T3

# Step 3 — Show DLQ is empty
docker exec -it rabbitmq rabbitmqadmin list queues --username=admin --password=password
# dead.letter.queue → 0

# Step 4 — Send 5 broken messages
for i in 1 2 3 4 5; do
  curl -X POST http://localhost:3001/api/test/broken-message
  echo "sent $i"
done

# Step 5 — Watch consumer terminal show:
# ☠️  [DLQ DEMO] Malformed JSON detected!
# ☠️  Message rejected → dead.letter.queue (total DLQ: 1)
# ☠️  Message rejected → dead.letter.queue (total DLQ: 2)
# ... up to 5

# Step 6 — Verify
docker exec -it rabbitmq rabbitmqadmin list queues --username=admin --password=password
# dead.letter.queue → 5 ✅

# Step 7 — Show in RabbitMQ UI
# http://localhost:15672 → Queues → dead.letter.queue → Get Messages
# Shows the broken payload: { this is broken json !!!
```
